关于 iOS 传值

  • 关于开发的那些事

利用通知中心传值(NSNotificationCenter)

1.接受传过来的值
[[NSNotificationCenter defaultCenter] addObserver:self  selector:@selector(demo:) name:@"hello" object:@"hello"];     
- (void)demo:(NSNotification *)noti
{
    NSLog(@"%@",noti.userInfo[@"hello"]);
}

2.发送通知并传值(利用字典可以传多个值)
NSString *str = @"one";
NSDictionary *dic = @{@"hello": str};
[[NSNotificationCenter defaultCenter]     postNotificationName:@"hello" object:@"hello" userInfo:dic];

问题:
当一个界面 push 到另一个界面时,你想在第一个界面传一个值给第二个界面,好,先在第一个界面注册通知并发送,然后在第二个界面再用 addObserver 的方法接受传来的值.一运行,结果不会显示.反而是用这个传值的方式,在第二个界面穿到第一个界面,诶,结果会有显示.
我有点不理解,出来的这种状况,可能没把这个通知机制,理解好,感觉是在这个 kvo 上的问题.

用 storyBoard 在界面之间传值

- (void)prepareForSegue:(UIStoryboardSegue *)segue sender:(id)sender
{
    UITableViewCell *cell = (UITableViewCell *)sender;
    NSIndexPath *indexPath = [self.tableView indexPathForCell:cell];
    id model = _students[indexPath.row];
    //组装模型
    NSString *name = [model valueForKey:kName];
    NSString *ID = [model valueForKey:kStuID];
    int age = [[model valueForKey:kAge] intValue];

    QYStudentModel *student = [QYStudentModel studentWithName:name ID:ID andAge:age];
    UIViewController *dstVC = segue.destinationViewController;
    [dstVC setValue:student forKey:@"student"];
}

心得:
这个方法用的着实方便,首先 segue 的能获得两个视图控制器,sourceViewController,destinationViewController.你想在一个界面传值给另一个界面的话,直接用 KVC 设置下个界面的一个属性,然后在下个直接使用这个属性,就达到了传值的效果,想想真的很 nice.这个显然的要比属性传值要简洁的很多.

今天的学习感受

在做些需求的时候,一开始,不必要的考虑到具体的代码,而是先把需求的整个流程逻辑的框图描绘出来,把逻辑方法一个一个的方法给罗列起来,简单的方法能先写上就写上,复杂点的先放着.有了这样的整体的这样的一个考虑之后,然后慢慢地具体到每一个的方法实现.这样的效率更高,而且还能很好地避免错误的出现.
