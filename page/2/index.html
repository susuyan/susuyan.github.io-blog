  1. 设置XML解析对象的代理

  2. 开始解析

NSXMLParser对象代理方法：

  • (void)parserDidStartDocument:(NSXMLParser *)parser

    当开始解析XML文档的时候，调用这个方法。通常在这个方法里，创建存储模型对象的数组

  • (void)parser:(NSXMLParser )parser didStartElement:(NSString )elementName namespaceURI:(NSString )namespaceURI qualifiedName:(NSString )qName attributes:(NSDictionary *)attributeDict

    当开始解析，遇到元素的开始标签的时候，调用这个方法。通常在这个方法里，创建模型对象或解析标签中的属性保存在模型对象中

  • (void)parser:(NSXMLParser )parser foundCharacters:(NSString )string

    当解析到xml标签的文本内容的时候，调用这个方法, 通常在这里先暂存解析到的文本内容

  • (void)parser:(NSXMLParser )parser didEndElement:(NSString )elementName namespaceURI:(NSString )namespaceURI qualifiedName:(NSString )qName

    当解析xml内容遇到结束标签的时候，调用这个方法。通常在这个方法里，需要将模型对象保存入数组中或把标签对应的文本内容解析出来，保存在模型对象中（KVC）

  • (void)parserDidEndDocument:(NSXMLParser *)parser

    当整个xml文档全部解析结束的时候，该方法会被调用

  • (void)parser:(NSXMLParser )parser parseErrorOccurred:(NSError )parseError

    当遇到解析错误时，该方法会被调用

Git常用操作

Git常用操作命令

1) 远程仓库相关命令

检出仓库：$ git clone git://github.com/jquery/jquery.git

查看远程仓库：$ git remote -v

添加远程仓库：$ git remote add [name] [url]

删除远程仓库：$ git remote rm [name]

拉取远程仓库：$ git pull [remoteName] [localBranchName]

推送远程仓库：$ git push [remoteName] [localBranchName]

2）分支(branch)操作相关命令

查看本地分支：$ git branch

查看远程分支：$ git branch -r

创建本地分支：$ git branch [name]
——注意新分支创建后不会自动切换为当前分支

切换分支：$ git checkout [name]

创建新分支并立即切换到新分支：$ git checkout -b [name]

删除分支：$ git branch -d [name]
—— -d选项只能删除已经参与了合并的分支，对于未有合并的分支是无法删除的。如果想强制删除一个分支，可以使用-D选项

合并分支：$ git merge [name]
——将名称为[name]的分支与当前分支合并

创建远程分支(本地分支push到远程)：$ git push origin [name]

删除远程分支：$ git push origin :heads/[name]

iOS 中的音频播放处理

使用 System sound service 播放 UI音效和调用振动效果

概述

iOS中最简单的播放声音就是播放系统声音,播放系统声音要用到System Sound Services,它是Audio Toolbox Framework的⼀一部分,它提供C语言风格接⼝用于播放短的声⾳音,如果对应的 设备⽀支持振动,还可以调⽤用振动效果。

使用 System sound service 的限制条件:

  • 音频文件播放长度不能大于30秒
  • 必须是线性的音频格式: PCM / IMA4 (IMA/ADPCM)
  • 文件格式必须是 .caf .aif .wav
    如果使用 AudioSerVicePlaySystemSound 函数,还有如下限制:
    • 音量大小调节,依赖于当前系统音量设置
    • 声音立即播放
    • 不支持循环播放,立体声和定位播放
    • 不支持同时播放,即一次只能播放一个声音
使用 AudioServicePlayAlertSound 函数
  • 除了播放声音文件外,还会调用振动功能

注意: 系统提供的警告声音和系统提供的用户界面的声音效果并不提供给您的应用程序.例如:使用￼kSystemSoundID_UserPreferredAlert常数作为参数传递给AudioServicesPlayAlertSound￼￼函数将不会有任何效果

iOS 中的绘图

绘图总结

负责绘制视图的方式是 UIView 的 drawRect: 默认情况下, drawRect: 不会做任何事情. UIView 的子类可以覆盖 drawRect:,完成自定义的绘图任务.
绘图的方法和函数都是源自 Core Graphics 框架中的 Quartz 2D

Quartz 2D

概述:

它是 ios 和 Mac 上的绘图引擎(在 kernel 之外的).在绘制图形的时候,我们只考虑使用它提供的 API(C语言接口)进行图形的各种绘制,而不用考虑图形的最终的渲染显示.

注意点:(结合官方文档,慢慢阅读和理解)

Graphics Contexts 图形上下文
Paths 路径 (Quartz 2D 是基于路径的绘制)
Color and Color Spaces 通过颜色空间创建颜色
Transforms 仿射变换(平移,旋转,缩放)也就是改变坐标系
Patterns 样式
Shadows 阴影
Gradients 颜色渐变 (线性渐变和径向渐变)
Transparency Layers 透明层
Data Management in Quartz 2D
Bitmap imagees and image Masks
Core Graphics Layer Drawing
PDF Document Creation, Viewing, and Transforming
PDF Document Parsing
PostScript Conversion
Text 绘制文本
Glossary

Example

Drawing a Line

画线之前,我们需要获得当前的View 的图形上下文 

CGContextRef context = UIGraphicsGetCurrentContext();

指定线宽

CGContextSetLineWidth(context, 2.0);

通过颜色空间创建颜色

CGColorSpaceRef colorspace = CGColorSpaceCreateDeviceRGB(); 
CGFloat components[] = {0.0, 0.0, 1.0, 1.0};
CGColorRef color = CGColorCreate(colorspace, components);

设置描边颜色

CGContextSetStrokeColorWithColor(context, color);

设置开始点

CGContextMoveToPoint(context, 30, 30);

设置结束点

CGContextAddLineToPoint(context, 300, 400);

开始绘制

CGContextStrokePath(context);
CGColorSpaceRelease(colorspace);
CGColorRelease(color);

简便写法


	- (void)drawRect:(CGRect)rect {
	    CGContextRef context = UIGraphicsGetCurrentContext();
	    CGContextSetLineWidth(context, 2.0);
	    CGContextSetStrokeColorWithColor(context, [UIColor
	       blueColor].CGColor);
	    CGContextMoveToPoint(context, 30, 30);
	    CGContextAddLineToPoint(context, 300, 400);
	    CGContextStrokePath(context);
	}

关于 iOS 传值

利用通知中心传值(NSNotificationCenter)

1.接受传过来的值
[[NSNotificationCenter defaultCenter] addObserver:self  selector:@selector(demo:) name:@"hello" object:@"hello"];     
- (void)demo:(NSNotification *)noti
{
    NSLog(@"%@",noti.userInfo[@"hello"]);
}

2.发送通知并传值(利用字典可以传多个值)
NSString *str = @"one";
NSDictionary *dic = @{@"hello": str};
[[NSNotificationCenter defaultCenter]     postNotificationName:@"hello" object:@"hello" userInfo:dic];

问题:
当一个界面 push 到另一个界面时,你想在第一个界面传一个值给第二个界面,好,先在第一个界面注册通知并发送,然后在第二个界面再用 addObserver 的方法接受传来的值.一运行,结果不会显示.反而是用这个传值的方式,在第二个界面穿到第一个界面,诶,结果会有显示.
我有点不理解,出来的这种状况,可能没把这个通知机制,理解好,感觉是在这个 kvo 上的问题.

多线程--第一篇

在学习多线程时,先简单的回忆下进程的一个概念

什么是进程

  • 狭义定义: 进程就是一段程序的执行过程
  • 广义定义: 进程是一个具有一定独立功能的程序关于某个数据集合的一次运行活动.它是操作系统动态执行的基本单元,在传统的操作系统中,进程既是基本的分配单元,也是执行单元.

进一步理解

(不要理解太深,有时间得回看一下之前学习的操作系统)

  • 进程是一个实体,每一个进程都有它自己的地址空间,一般情况下,包括文本区域(写的以一些代码),数据区域(各种变量)和堆栈.
  • 进程是一个”执行中的程序”
